# Playtests — The Player Council

**Owner:** UX Researcher · **Feeds:** [`DESIGN_DEBT.md`](../DESIGN_DEBT.md), [`UX-GUIDELINES.md`](../UX-GUIDELINES.md), [`RISK-REGISTER.md`](../RISK-REGISTER.md)

Professional studios obsess over watching real people play. Almost no indie
developer does it systematically — so doing it is an edge. Each session is one
file (`001.md`, `002.md`, …) using the template below.

**The one rule:** where a player hesitates, the *design* is at fault, not the
player. Record what you saw, not what you wish you'd seen.

## Who to recruit (match the real audience)

Entry-tier Android, on mobile data, across ages and home languages (isiXhosa and
English first). Include low-literacy testers — the first minute must be winnable by
someone still building reading skill ([`ACCESSIBILITY.md`](../ACCESSIBILITY.md)).
Test the **forwarded-WhatsApp-link cold open**, not a pre-installed build — that's
the real first touch.

## What to capture (watch the first 60 seconds silently)

- Where they were **confused**.
- Where they **smiled** (the magic moments — [`MAGIC.md`](../MAGIC.md)).
- Where they **quit**.
- Their **favourite** and **worst** moment.
- Whether they hit a **win in under 60 seconds** (Principle 3).
